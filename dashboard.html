BolsterMed Candidates Page – Final MVP Refinements Goal: Elevate the Candidates page from a clean list into a polished, high-utility pipeline view with clear prioritization, strong visual cues, and improved usability. ⸻ 1. Emphasize “Ready for Review” Rows This is the highest-value candidate state and should stand out more. Enhance by: • Adding a subtle light green background to the row OR • Adding a left border highlight OR • Slightly bolding the candidate name Reason: Draws immediate attention to candidates ready for decision-making. ⸻ 2. Improve “Updated” Column (Time-Based Context) Current format uses static dates, which are less intuitive. Update to: • “Updated 6d ago” OR • “Last activity 6d ago” Optional: • Keep full date as secondary (tooltip or smaller text) Reason: Users think in relative time, not calendar dates. ⸻ 3. Make Entire Rows Clickable Currently only the “Open” button is clickable. Update: • Entire row clickable → opens candidate detail page • Keep “Open” as explicit CTA Reason: Reduces friction and improves navigation speed. ⸻ 4. Add Hover States (UI Polish) • Row hover → subtle background change • Cursor changes to pointer Reason: Improves interactivity and makes UI feel more responsive. ⸻ 5. Refine Progress Display (Minor Improvement) Current format: “1 of 7” Optional improvements: • “1 / 7” • “1 of 7 refs” • Future: “6 remaining” Reason: Slightly improves clarity and readability. ⸻ 6. Optional: Add Sorting Indicators • Add subtle sort icons (↑ ↓) on column headers Reason: Prepares for scalability and enhances usability with larger datasets. ⸻ 7. Minor Visual Enhancements • Slightly increase thickness of progress bars for readability • Improve contrast of status pills (clearer differentiation between states) ⸻ Core Principle Each row should clearly communicate: Who this is, where they are in the process, and whether I need to act ⸻ Summary • Maintain current structure (strong foundation) • Enhance visual prioritization (Ready for Review) • Improve usability (clickable rows, hover states) • Add clarity (time-based updates)
